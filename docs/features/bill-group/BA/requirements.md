# 账单分组（Bill Group）

## 背景与动机

当前系统分类维度：**账本**（个人/AA）→ **分类**（餐饮/交通/购物等）。用户希望对个人账本内的消费进行更进一步的事件/项目级聚合，例如「云南旅游」「浙江旅游」「新房装修」等，方便事后快速查看某个事件的消费总额。

## 用户故事

1. **默认分组**：每笔流水默认计入「日常消费」账单分组，用户无需额外操作即可正常使用
2. **新建时选择**：在新建流水 Modal 中可主动选择账单分组
3. **事后修改**：在账单详情 Sheet 中可更改某笔流水的账单分组
4. **批量移动**：在账单列表中可批量选中流水，移动到指定分组
5. **列表展示**：账单列表金额下方的 OCR 来源文字替换为账单分组名称
6. **分组统计**：统计 Tab 新增账单分组消费金额排名，一眼看出各分组总消费

## 功能范围

### 做什么

- 新增 `BillGroup` 实体（SQLite 表 + Swift 模型）
- 系统初始化时创建「日常消费」预设分组（isDefault = true，不可删除）
- `Record` 表新增 `bill_group_id` 列（NOT NULL，历史数据迁移到「日常消费」）
- 账单分组 CRUD：创建、重命名、删除（软删除，预设分组不可删）
- 新建流水 Modal 增加账单分组选择行
- 账单详情 Sheet 可修改账单分组
- 账单列表批量移动流水到指定分组
- 账单列表金额下方展示账单分组名（替换 OCR 来源文字）
- 统计 Tab Hub 新增「账单分组」卡片
- 新增 `StatsBillGroupView` 分组排行详情页
- `StatsMainView`（本月统计）分类构成下方新增账单分组概览区块
- `StatsViewModel` 新增按账单分组聚合的派生数据

### 不做什么

- AA 账本不支持账单分组（仅个人账本）
- 不改变现有 Category（分类）逻辑
- 账单分组不参与飞书同步字段映射（本次不做）
- 不改变 AA 占位流水的展示逻辑

## 验收标准

1. 新建流水时，默认选中「日常消费」；用户可切换到其他分组
2. 账单列表每行金额下方显示账单分组名（非"本地OCR"/"手动"等来源文字）
3. 账单详情可修改分组，修改后列表立即刷新
4. 批量模式下可选择多笔流水移动到某分组
5. 统计 Tab「账单分组」卡片显示本月支出最高分组名 + 金额
6. 分组详情页按金额降序排列，显示金额/笔数/占比
7. 本月统计页展示 Top5 账单分组 + 金额柱状图
8. 预设「日常消费」分组不可删除
9. 所有更改经过 TDD（80%+ 覆盖）

## 依赖与风险

- **依赖**：`Record` 表 schema 变更需要新增 migration（v7），历史数据 `bill_group_id` 回填
- **依赖**：`SQLiteRecordRepository` 查询需支持按 `billGroupId` 筛选
- **风险**：schema migration 在用户已有大量数据时需充分测试回填逻辑
- **风险**：`RecordRow.sourceText` 替换后需确认 AA 占位流水不受影响（仍保持原有展示逻辑）
